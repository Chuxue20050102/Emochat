# EmoChat 部署排障简记（2026-04-27）

## 1. 目标
- 让前端真机可访问后端
- 让团队可通过公网地址联调
- 修复 AI 聊天返回兜底文案问题

## 2. 主要问题与结论
1. `docker compose build` 报 `no configuration file provided`
- 原因：项目根目录缺少 `docker-compose.yml`
- 处理：补充 Compose、Backend Dockerfile、.dockerignore

2. Natapp 域名可访问异常（`Tunnel not found` / `Invalid Hostname`）
- 原因：
  - natapp 进程被关闭（按了 `Ctrl+C`）
  - 隧道配置端口与后端端口不一致（80/8080 vs 8000）
  - Clash 规则/fake-ip 干扰解析与连通
- 处理：
  - 确认隧道本地端口 `8000`
  - 保持 natapp 前台运行
  - 对 natapp 相关域名做直连规则

3. 登录成功但 AI 聊天兜底
- 现象：后端日志持续 `[AI Service Error] Connection error.`
- 结论：主要是上游模型网络连通问题，不是前端路由问题

4. 本机后端启动期间出现 500
- 现象：`/api/auth/guest` 返回 500
- 原因：SQLite 报 `disk I/O error`
- 结论：SQLite 在当前环境不稳定，切 MySQL 是正确方向

5. 切 MySQL 后启动报 `No module named 'pymysql'`
- 原因：缺少 MySQL Python 驱动
- 处理：安装 `pymysql`

## 3. 已完成改动
- 新增：
  - `docker-compose.yml`
  - `backend_fastapi/Dockerfile`
  - `backend_fastapi/.dockerignore`
- 修改：
  - `backend_fastapi/database.py`（支持 `EMOCHAT_DATABASE_URL`）
  - `backend_fastapi/requirements.txt`（加入 `pymysql`）
  - `frontend_vue/config/index.js`（联调期间改为 natapp 域名）

## 4. 当前建议启动方式（开机后）
1. 启动基础服务
- `docker compose up -d mysql redis`

2. 启动后端（本机 venv）
- `cd backend_fastapi`
- `$env:EMOCHAT_DATABASE_URL="mysql+pymysql://emochat:emochat123@127.0.0.1:3306/emochat?charset=utf8mb4"`
- `python -m uvicorn main:app --host 0.0.0.0 --port 8000`

3. 启动 natapp（保持窗口不关闭）
- `./natapp.exe -log=stdout -authtoken=<你的token>`
